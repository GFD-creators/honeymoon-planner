# 行き先の削除（都市・見どころ）設計仕様書

作成日: 2026-06-07
ステータス: 設計確定（ユーザー承認済み）
対象アプリ: ハネムーン・プランナー（既存。行き先連動 `2026-06-07-inline-create-and-place-link-design.md` の続き）

## 1. 目的 / 背景

行き先タブで不要な都市カード（例：ローマ）や個々の見どころを削除できるようにする。
旅程にリンク済みの見どころを削除する場合は、リンク先の旅程予定も一緒に削除して取り残しを防ぐ。

## 2. 確定要件

- **都市カードの見出しに削除ボタン**（🗑）→ 確認ダイアログ → その都市を `cities` から削除
  - その都市の見どころに旅程リンク（`eventId`）があれば、リンク先の `events` も併せて削除
  - 確認文言: 「『{都市名}』を削除しますか？関連する旅程の予定も削除されます。」
- **各見どころの行に削除ボタン**（✕）→ その見どころを当該都市の `sights` から削除
  - リンク無し: 確認なしで即削除
  - リンク有り（`eventId` が実在する予定を指す）: 確認ダイアログ後、見どころとリンク先 `events` を削除
  - 確認文言: 「この見どころとリンクした旅程の予定も削除しますか？」

## 3. スコープ外（YAGNI）

- 都市の新規追加・名前/滞在日数の編集・並び替え
- 削除のアンドゥ

## 4. アーキテクチャ / ファイル構成

```
js/
├ link.js     — 変更。linkedEventIds(sights) を追加（純粋・テスト対象）
├ places.js   — 変更。都市見出しに削除ボタン、見どころ行に✕、削除処理
css/styles.css — 追加。削除ボタンのスタイル
tests/
└ link.test.js — 変更。linkedEventIds のテストを追加
```

`store.js` / `editor.js` / `weekview.js` / `timeline.js` / `week.js` / `drag.js` / `seed.js` は変更しない。

## 5. link.js に追加する純粋関数

```
linkedEventIds(sights) -> string[]
  sights 配列の各要素の eventId のうち truthy なものを配列で返す。
  例: linkedEventIds([{eventId:'e1'},{},{eventId:'e3'}]) -> ['e1','e3']
```

## 6. places.js の変更

### 都市削除
- 都市カード見出し（`.day-head`）に削除ボタン `.city-delete`（🗑）を追加
- クリック→ `confirm('「{city.name}」を削除しますか？関連する旅程の予定も削除されます。')`
- OK の場合 `update`:
  - 対象都市の `linkedEventIds(city.sights)` を取得
  - `s.events` からその id を持つ予定を全削除
  - `s.cities` から当該都市（name 一致）を削除

### 見どころ削除
- 各見どころ行（`.sight`）の `.sight-actions` に削除ボタン `.sight-delete`（✕）を追加
- クリック時:
  - その見どころに実在のリンク予定があるか（`findLinkedEvent`）を判定
  - 有り: `confirm('この見どころとリンクした旅程の予定も削除しますか？')`、OK なら予定と見どころを削除
  - 無し: 即削除
- 削除の `update`:
  - リンク予定があれば `s.events` から該当 id を削除
  - 当該都市の `sights` から index で削除

## 7. CSS 追加

- `.city-delete`: 見出し右端の小さな削除ボタン（控えめな色）
- `.sight-delete`: 見どころ行右端の小さな✕

## 8. テスト方針

- **link.js**: `linkedEventIds` の単体テスト（truthy のみ収集、空配列、未設定）
- **ブラウザ実機（Playwright）**:
  - ローマカードの🗑→確認→カードが消える
  - 見どころの✕→消える（リンク無しは即時）
  - 旅程に追加済みの見どころを✕→確認→見どころと旅程予定の両方が消える
  - 都市削除でリンク予定も消える
  - 変更が Firestore 同期で2タブ目に反映

## 9. 成功基準

- 行き先タブで都市カード（ローマ等）を確認付きで削除できる
- 個々の見どころを削除できる（リンク有りは確認＋予定も削除）
- 都市削除時にリンク予定が旅程から取り残されない
- 既存データがそのまま動作する
