# 週グリッドの表示幅拡張 設計仕様書

作成日: 2026-06-07
ステータス: 設計確定（ユーザー承認済み）
対象アプリ: ハネムーン・プランナー（既存。週タブの見た目調整）

## 1. 目的 / 背景

週グリッドが全タブ共通の幅560px制限内にあり、広い画面（PC/タブレット）でも横幅が狭く
7列を見るのに横スクロールが要る。週タブのときだけ表示領域を広げ、列を見やすくする。

## 2. 確定要件

- **週表示のときだけ** `#view` の最大幅を `min(96vw, 1100px)` に拡張する
- スマホ（狭い画面）は 96vw により従来通り（横スクロールで全曜日を閲覧、レイアウト崩れなし）
- リスト表示・行き先・予算タブは従来の **560px** のまま（読みやすさ優先）
- ロジック変更なし

## 3. スコープ外（YAGNI）

- 列幅・フォントサイズの再調整（今回は表示領域の最大幅のみ）
- 他タブの幅変更

## 4. アーキテクチャ / ファイル構成

```
css/styles.css — 追加。`.view.wide { max-width: min(96vw, 1100px); }`
js/timeline.js — 変更。週表示時に root（#view）へ `wide` クラス付与、リスト表示で除去
js/app.js      — 変更。各タブ描画前に `wide` クラスをリセット（既定は narrow）
```

`weekview.js` / `places.js` / `budgetView.js` / `store.js` などロジックは変更しない。

## 5. 実装方針

- `index.html` の `<main id="view" class="view">` が表示領域。既存 CSS は `.view { max-width: 560px; margin: 0 auto; }`
- `.view.wide` で 560px を上書きして拡張
- `js/app.js` の `render()` 冒頭で `viewEl.classList.remove('wide')` を実行（タブ切替・再描画のたびに既定で narrow に戻す）
- `js/timeline.js` の `setView(v)` で `root.classList.toggle('wide', v === 'week')`
  - renderTimeline は render() から同期的に呼ばれるため、週表示中は毎回 wide が再付与される（行き先/予算では renderTimeline が呼ばれないので wide は付かない）

## 6. テスト方針

- ロジック変更なし＝単体テスト追加なし（既存 39 件が緑のまま）
- **ブラウザ実機（Playwright）**:
  - 広いビューポート（例 1280px）で週表示にすると `#view` が約1100px幅になり、7列がスクロールなしで見える
  - リスト/行き先/予算に切替えると `#view` が560pxに戻る（wide クラスが外れる）
  - 狭いビューポート（390px）では従来通り横スクロールで全曜日が見える

## 7. 成功基準

- PC/タブレットで週タブを開くと、カレンダーが横に広がり7列が一望できる
- 他タブ・スマホ表示は従来通り
- 既存機能（ドラッグ・空きマス作成・編集・ページ移動）が壊れない
