# ハネムーン・プランナー 設計仕様書

作成日: 2026-06-07
ステータス: 設計確定（ユーザー承認済み）

## 1. 目的 / 背景

新婚旅行（2026/9/12〜9/23 ヨーロッパ周遊）のスケジュール・行き先・予算を
楽しく入力でき、時差やフライトを含む1日の流れが見える化されるダッシュボード。
夫婦2人がLINEでURLを共有し、同時に編集できることを必須要件とする。

## 2. 確定要件

- **モバイル優先**のレスポンシブWebアプリ
- **ポップなUI**（参考: shiorisu.com のような明るく親しみやすい配色）
- **カレンダー縦タイムライン**（参考: note.com 旅程アプリのような1日の流れ表示）
- **時差対応**: 現地時刻と日本時刻を併記
- **フライト/乗継の可視化**: 出発→到着、所要時間、乗継待ち時間
- **予算可視化**: 内訳グラフ、合計、1人あたり、ユーロ換算、予算 vs 実績
- **共同編集**: 2人がリアルタイムで同時編集
- **公開**: GitHub Pages、URLをLINEで共有

## 3. 技術構成

- **フロントエンド**: バニラ HTML / CSS / JavaScript、単一ページ、ビルド不要
  - グラフ描画は軽量に自前 SVG もしくは Chart.js（CDN）
- **データ同期**: Firebase Firestore（CDN読み込み、`onSnapshot` でリアルタイム購読）
- **認証**: なし。推測されにくいランダムな `tripId` を URL クエリ `?trip=xxxx` に持たせ、
  URL を知っている人のみアクセス可能（夫婦2人のみ共有前提）
- **ホスティング**: GitHub Pages
- **Firebase初回設定**: ユーザーと一緒に実施（無料プロジェクト作成＋設定キー貼り付け、手順を案内）

### 採用しなかった案
- React + Vite: 構造化されるがビルドが必要。手軽さ優先で不採用。
- Notion等の既製ツール: ポップUIの自由度が出ないため不採用。

## 4. 画面設計（3タブ・下部タブバー）

### ① 📅 タイムライン（メイン）
- 日付ごとの縦スクロール・タイムライン
- 各予定をカード表示。種別をアイコン＆色分け:
  - 🛫 移動（フライト/電車） / 📸 観光 / 🍽 食事 / 🏨 宿泊 / 🛍 その他
- 時刻表示: 現地時刻を大きく、日本時刻を小さく併記（例: ロンドン 14:20 / 🇯🇵 22:20）
- **フライトカードは特別表示**: 出発地→到着地、所要時間、乗継待ち（例: 韓国 3h40m 待ち）
- カードタップで編集、＋ボタンで予定追加・削除

### ② 🗺 行き先＆やること
- 都市カード: ロンドン 2日 / パリ 2.5日 / ヴェネチア 2.5日
- 都市ごとの見どころチェックリスト（訪問済みにチェック）
  - ロンドン: 大英博物館 / バッキンガム宮殿 / ビッグベン
  - パリ: エッフェル塔 / エトワール凱旋門(シャンゼリゼ通り) / ルーブル美術館
  - ローマ・ヴェネチア: バチカン市国 / ベネチア

### ③ 💰 予算
- ドーナツグラフで内訳表示
- 合計 ¥1,918,410 / 1人あたり ¥959,205 / €換算（@180円）
- 「予算」「実績」の2列。実績入力で残額が見える
- 内訳項目（初期データ）:
  - 日本→イギリス 280,400（アシアナ航空）
  - イギリス→パリ 50,000（ユーロスター、2時間半）
  - パリ→ヴェネチア 100,000（エールフランス/イージージェット、1時間半）
  - ヴェネチア→日本 471,010（ITA AIRWAYS）
  - ホテル 270,000 / 食事 410,000 / 交通 30,000
  - 観光チケット 42,000 / Wi-Fi・eSIM 35,000 / 海外保険（未定）/ 前撮り 230,000

## 5. データ構造（Firestore）

```
trips/{tripId}
  ├ meta: { title, startDate: 2026-09-12, endDate: 2026-09-23,
  │         rates: { EUR: 180, GBP: 210 }, updatedAt }
  ├ events: [
  │   { id, date, time, city, tz, type, title, note,
  │     meals: { breakfast, lunch, dinner },
  │     flight?: { from, to, durationMin, layoverMin } }
  │ ]
  ├ cities: [ { name, nights, days, sights: [{ name, done }] } ]
  └ budget: [ { category, planned, actual, note } ]
```

- 時刻は「現地時刻 + タイムゾーン文字列」で保持し、表示時に日本時刻を算出
- タイムゾーン: 日本 JST(UTC+9) / 韓国 KST(UTC+9) / ロンドン BST(UTC+1) /
  パリ・ヴェネチア・ローマ CEST(UTC+2)（2026年9月は夏時間）

## 7. 共同編集の動作

- `onSnapshot` で `trips/{tripId}` を購読。片方の編集が即もう片方に反映
- 競合解決: Last-Write-Wins（2人のみのため十分）
- 各書き込みで `meta.updatedAt` を更新

## 8. スコープ外（YAGNI）

- ユーザー認証 / アカウント管理
- 複数旅行の管理（今回は1旅行のみ。tripIdで拡張余地は残す）
- オフライン編集の競合マージ
- 通貨レートの自動取得（手入力）
- 地図API連携（見どころはテキストチェックリストで十分）

## 9. 成功基準

- スマホで開いて1日のスケジュールが時系列で見える
- 現地時刻と日本時刻が両方分かる
- フライト・乗継待ちが直感的に分かる
- 予算の内訳と残額が一目で分かる
- 妻にURLを送って、2人で同時に編集でき、変更が即同期される
