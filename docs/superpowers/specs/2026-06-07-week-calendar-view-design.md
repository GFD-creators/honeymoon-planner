# 週カレンダー表示＋日時ピッカー編集 設計仕様書

作成日: 2026-06-07
ステータス: 設計確定（ユーザー承認済み）
対象アプリ: ハネムーン・プランナー（既存。`2026-06-07-honeymoon-planner-design.md` の続き）

## 1. 目的 / 背景

旅程の予定を「日付・時間が簡単に変更でき」、かつ Google カレンダーのような
**週単位（日〜土）グリッド**で1日の流れと予定の移動が見えるようにする。

## 2. 確定要件

- 「旅程」タブに**表示切替トグル [リスト｜週]** を追加。既存のリスト表示は残す
- **週グリッド表示**：日〜土の7列＋左に時刻軸。前の週/次の週ナビゲーション
- **時刻あり予定**は時間位置にブロック表示、**終日予定**は上部「終日」行に表示
- **スマホ優先**：狭い画面では7列を横スワイプでスクロール（時刻軸は左固定）。PCでは7列フィット
- **タップ編集**：予定をタップ→編集シートで日付（日付ピッカー）・開始/終了時刻（時刻ピッカー）・内容・種別・削除を編集。`prompt()` 連打を廃止し、リスト表示・週表示の両方でこの編集シートを使う
- ドラッグ＆ドロップは**採用しない**

## 3. スコープ外（YAGNI）

- ドラッグ＆ドロップでの予定移動
- 同時刻に重なる予定の高度なレイアウト（簡易な横並びのみ。3件以上の重なりは等幅分割で割り切り）
- 繰り返し予定、通知、Googleカレンダー連携

## 4. アーキテクチャ / ファイル構成

既存の描画モジュール分離パターンに従う。週の計算ロジックは純粋関数として切り出し単体テスト。

```
js/
├ week.js        — 新規。週・時刻の純粋計算（テスト対象）
├ editor.js      — 新規。編集シート（モーダル）。リスト/週の両方から使う
├ weekview.js    — 新規。週グリッド表示の描画
├ timeline.js    — 変更。リスト表示。prompt編集を廃止し editor.js を呼ぶ。＋ビュー切替トグル
├ app.js         — 変更なし想定（timeline が旅程タブ全体を描画）
├ seed.js        — 変更なし（endTime は任意。既存データは未設定でOK）
└ store.js       — 変更なし（update API をそのまま利用）
css/styles.css   — 追加。週グリッド・編集シートのスタイル
tests/
└ week.test.js   — 新規。week.js の単体テスト
```

責務:
- **week.js**（純粋）: 週の起点計算・日付列生成・時刻→位置変換・予定の日別グルーピング
- **editor.js**: 1件の予定を編集するモーダルを開き、保存/削除時に `store.update` を呼ぶ。描画から独立
- **weekview.js**: state を受け取り週グリッドDOMを描画。予定タップで editor を開く
- **timeline.js**: 旅程タブのコンテナ。トグル状態を持ち、リスト or 週ビューを描き分ける

## 5. データモデル変更

予定 `event` に**任意フィールド `endTime`**（'HH:MM'）を追加。
- 時刻あり予定で `endTime` 未設定なら、グリッド上は開始から既定60分のブロックとして描画
- 既存シードデータは `endTime` 未設定のまま動作する（後方互換）

## 6. week.js のインターフェース（純粋関数）

```
weekStartSunday(dateStr) -> 'YYYY-MM-DD'
  指定日が属する週の日曜日を返す。

weekDays(weekStartStr) -> ['YYYY-MM-DD' x7]
  日曜起点の7日分の日付配列。

addWeeks(weekStartStr, n) -> 'YYYY-MM-DD'
  週起点を n 週ずらす（前/次の週ナビ用）。

timeToMinutes(timeStr) -> number        // '14:20' -> 860
minutesToTop(min, opts) -> number       // グリッド開始時刻からのpxオフセット
blockHeight(startStr, endStr, opts) -> number  // ブロック高さpx。endなし時は既定60分

splitTimedAllDay(events) -> { timed: [...], allDay: [...] }
  時刻あり/終日（time が空）に分割。
```

`opts` は `{ startHour, hourHeight }`（例: startHour=5, hourHeight=48）。

## 7. UI 仕様

### 表示切替トグル
旅程タブ上部に2ボタン `[リスト | 週]`。既定は**リスト**（スマホで開く想定が多いため）。
選択状態は `localStorage`（キー `hp-timeline-view`）に保存し次回も維持。

### 週グリッド
- 左に時刻軸（既定 **午前5時〜翌2時** をカバー、`startHour=5`, 21時間分）。hourHeight 48px
- 上部に曜日ヘッダ（日〜土＋日付。旅行期間内の日付を強調）と「終日」行
- 「‹ 前の週」「今日/旅程へ」「次の週 ›」ナビ。初期表示は**出発日 2026-09-12 を含む週**
- 時刻あり予定: 種別色のブロック。時刻＋内容を表示。タップで編集シート
- 終日予定: 「終日」行にチップ表示。タップで編集シート
- 旅行期間外の日付列も表示されるが薄く（予定が無いだけ）
- スマホ: グリッドを `overflow-x:auto` で横スクロール、各日列 `min-width` 約 92px、時刻軸は `position:sticky; left:0`

### 編集シート（editor.js）
画面下から出るモーダル（オーバーレイ＋シート）。フィールド:
- 内容（text）
- 種別（select: 移動/観光/宿泊/食事/その他 = flight/sightseeing/hotel/food/other）
- 日付（`<input type=date>`）
- 開始時刻（`<input type=time>`、空可＝終日）
- 終了時刻（`<input type=time>`、空可）
- ボタン: 保存 / 削除 / キャンセル
保存で `store.update`（該当 event を書き換え）、削除で配列から除去。新規追加（＋ボタン）も同じシートを空値で開く。

## 7.1 既存リスト表示の変更
`timeline.js` の `editEvent`/`addEvent` の `prompt()` 連打を廃止し、`editor.openEditor(event|null)` を呼ぶ。
表示内容（カード）は現状維持。

## 8. テスト方針

- **week.js**: 単体テスト（node --test）。週起点（週またぎ・月またぎ）、7日生成、addWeeks、timeToMinutes、minutesToTop、blockHeight（end有/無）、splitTimedAllDay
- **weekview.js / editor.js / トグル**: ブラウザ実機確認（Playwright）。週移動・予定タップ→編集→保存→反映、終日/時刻ありの振り分け、スマホ幅で横スクロール、編集後 Firestore 同期

## 9. 成功基準

- 旅程タブで [リスト|週] を切り替えられ、選択が保持される
- 週グリッドが Google カレンダー風に7列で表示され、前後の週に移動できる
- 予定をタップすると編集シートが開き、**日付・時刻をピッカーで変更**でき、グリッド/リスト/予算に反映される（＋ Firestore 同期で妻側にも反映）
- スマホ幅でもグリッドが破綻せず横スクロールで全曜日を見られる
- 既存のシードデータ（endTime 無し）がそのまま正しく表示される
